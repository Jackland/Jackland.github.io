2. elastic 基础 介绍

  • document 文档数据 用户存储es 的数据文档（数据库）
  • index 索引 （列表mysql table表）
  • type 索引中的数据类型
  • field 字段 文档是属性

    3，rest api elstic集群对外提供的api

    • 交换方式 curl命令行
    • kibana devtools(安装kibana图形界面)

      4,索引api 新建 更新 删除索引配置

    • 创建索引 创建索引 更新索引 删除索引 put /test_index
    • 创建文档 put /test_index/doc/1 批量创建文档 post _bulk

5，正排索引 和倒排索引

  • 正排索引 文档id 文档内容和单词的关联关系

  • 倒排索引 单词和文档id的关联关系 单词词典是倒排索引的组成部分（B+tree）

6，分词

  • 分词 文本分析，将文本转化单词
  • 分词 组成 1,character 2,tokenzer 3, token filters

7,analyze api（测试分词结果的）

  • 调试 测试结果的
  • 自己构建分词器 进行测试 预定义分词器有很多种类参数 pattern analyze... keyword analyze....language analyze(语言分词)..

8 ，中文分词

常见中文的分词系统。ik ;jieba；高级的分词系统（hanlp 分词系统 4，thulac ） 中文分词 1，Ik 实现中英文单词切分 2，jieba python 分词系统 繁体和分词

  • character filter
  • tokenizer (将原始文本安装规则切分为单词)（letter,standard,whitespace...）

自定义mapping

; mapping 类似数据库的表字段定义 （int string） 定义倒排索引的配置 比如索引 ，记录position(字段类型 一旦新建 不予许修改) index 控制当前字段是否是索引，默认是true false不记录 ，及不可搜索

；数据类型 字符类型 text keyword 数值型 long .integer short byte double,float... 日期类型 date 布尔类型 boolen 复杂类型 array 地理位置; 专用类型

；dynamic mapping 自动识别创建文档字段类型
