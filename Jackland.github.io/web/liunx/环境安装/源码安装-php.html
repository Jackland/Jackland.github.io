源码安装php

源码安装php 7.2.19

源码安装的痛苦

    -- 安装切记开启mysqli 和 mysqlnd （我掉进去的坑）
    -- 操作方法 --enable-pdo 
             --with-pdo-mysql
             --enable-mysqlnd 
              --with-mysqli
    -- 还要查看 php.ini 的位置 放置正确，（坑）
    --总结 因为 php 的加载的扩展 （有一些扩展提前安装编译最好，省的回头再装 麻烦）

1-先下载包

2-安装源码安装的扩展 具体需要哪些安装哪些 如gcc 和 g++ 的环境

3-安装依赖

yum install -y gcc
yum install -y gcc-c++

yum -y install libxml2
yum -y install libxml2-devel 
(ps：说明一下2个库的区别  lib,lib-dev，前一个是程序运行时需要的库，后一个是编译源码时需要的库，只不过一个是运行库，一个是开发库)

4-执行configure文件

./configure --prefix=/usr/local/php --with-config-file-path=/usr/local/php --with-curl --with-freetype-dir --with-gd --with-gettext --with-iconv-dir --with-kerberos --with-libdir=lib64 --with-libxml-dir --with-mysqli --with-openssl --with-pcre-regex --with-pdo-mysql --with-pdo-sqlite --with-pear --with-png-dir --with-jpeg-dir --with-xmlrpc --with-xsl --with-zlib --with-bz2 --with-mhash --enable-fpm --enable-bcmath --enable-libxml --enable-inline-optimization --enable-gd-native-ttf --enable-mbregex --enable-mbstring --enable-opcache --enable-pcntl --enable-shmop --enable-soap --enable-sockets --enable-sysvsem --enable-sysvshm --enable-xml --enable-zip


./configure --prefix=/usr/local/php --with-config-file-path=/usr/local/php  --with-mysqli --with-pdo-mysql  --enable-fpm --enable-bcmath --enable-libxml --enable-inline-optimization --enable-gd-native-ttf --enable-mbregex --enable-mbstring --enable-opcache --enable-pcntl --enable-shmop --enable-soap --enable-sockets --enable-sysvsem --enable-sysvshm --enable-xml --enable-zip


 ps 这里注意  如果开启太多库和扩展 导致 make & make install 失败

参数解释
--prefix=PATH 安装到哪个目录里面
--bindir=DIRECTORY   
    为可执行程序声明目录，缺省是   EXEC-PREFIX/bin，   通常也就是   /usr/local/pgsql/bin．

 --with-curl 需要安装的php 包（例如curl gd库 等）

 --enable-syslog   开启安装的功能(例如 fpm zip pcntl 等php扩展) 

 --with-config-file-path=/usr/local/php  （这样的话php就回去指定的目录下读取php.ini配置文件，如果不加这个参数默认位置就是php安装目录下的lib目录，具体也可以在phpinfo()输出界面查看，如果php.ini放到其他位置，php读取不到，那么所有的配置修改后都是不生效的，这点要注意）

5-make

6-make install （或者 5 6 合在一起make && make install）
