第2节：swoole进阶学习

swoole 学习笔记

1，快速查询命令一栏 ps -aft| grep tcp.php 查看进程树状图 pstree -p 9501 查看进程树 netstat -anp |grep 8811 查看进程

2，swoole 有多少种server tcp，udp,http_server,webSocket通常这几种 其中websocket使用较多。

3，reactor_id 线程id 为什么等于0 ？ 进程没有被守护 导致每次新开客户端都是永远一个进程（理解为）

4，swoole 的进程是什么意思 服务端 ==》 $serv->set([ 'worker_num' => 4 , // worker进程数 cpu 1-4 'max_request' => 10000, ]); 设置的进程数时 、四个相关进程 一个master进程

5，task 异步任务 应用场景 ：处理响应时间很长的异步任务,不会影响服务端响应客户端时间。 相关函数 ：onTask() onFinsh() 6，定时器 swoole_timer_tick(); swoole_timer_after(); 7，process php脚本使用时候 注意参数的使用 1，设置进程数要结合实际情况不能过多浪费 也不能过少影响效率 2，管道 是进程与进程之间的桥梁 进行通信。

8、可以使用 set 函数快速自定义目录根路径 做到开启服务之间访问根目录的文件 //设置加载静态资源 $http->set( [ 'worker_num' => 4, 'enable_static_handler' => true, 'document_root' => '/mnt/hgfs/vmwww/swoole/thinkphp/public/static' ] );

9、Reactor、Worker、Task三种角色分别的职责是： 可以理解为reactor就是nginx，worker就是php-fpm。 reactor线程异步并行地处理网络请求，然后再转发给worker进程中去处理。reactor和worker间通过IPC方式通信。swoole的reactor，worker，task_worker之间可以紧密的结合起来，提供更高级的使用方式。 一个更通俗的比喻，假设Server就是一个工厂，那reactor就是销售，帮你接项目订单。而worker就是工人，当销售接到订单后，worker去工作生产出客户要的东西。而task_worker可以理解为行政人员，可以帮助worker干些杂事，让worker专心工作。

10、__call 魔术方法 实例化类的方法

11、使用swoole监控swoole的服务运行情况防止服务中断 影响服务。 思路可以写监控脚本 使用swoole 的定时器监控内容

12、 swoole 可以设置进程名称
平滑重启 swoole 和nginx 结合使用需要通过nginx服务 转发到swoole 服务器 使用nginx转发到多台swoole 上去 比较耗nginx cpu资源
