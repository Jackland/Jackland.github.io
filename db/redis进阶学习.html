第2节：redis优化

基本介绍

  1. 单线程 数据存储于内存中 支持数据可持久化

  2. redis 单线程采取多路复用的方法 增加其并发访问量

  3. redis 支持2种持久化机制 RDB和AOF；rdb 是将数据存储到硬盘中去 aof 是存储到log 文件中（数据过大 增加重写机制），常用rdb

秒杀业务 队列相关

redis 写进内存 数据可持久化

秒杀限制 将数据存入redis (uid date等用户信息) 时间存入为微秒时间戳，防止一秒内提交过多 思考业务场景 秒杀的数量 秒杀时间（微秒），并发量，请求量
